## 🎨 日志图标说明

| 图标 | 含义 | 使用场景 |
|------|------|----------|
| 🚀 | 任务调度开始 | `dispatchTaskToAgent` 被调用 |
| 📋 | 任务信息 | 显示任务基本信息 |
| 📝 | 详细信息 | 任务描述、参数等 |
| 🔍 | 检查/查询 | 检查 Agent 状态 |
| ⏳ | 等待中 | Agent 忙碌，需要等待 |
| ⏸️ | 暂停/排队 | 任务进入等待队列 |
| ✅ | 成功/就绪 | Agent 空闲，可以调度 |
| ❌ | 失败/错误 | Agent 离线或超时 |
| ⚠️ | 警告 | Agent 离线提醒 |
| 📊 | 状态更新 | 任务状态变化 |
| 👋 | 任务开始 | Agent 开始执行任务 |
| 🔨 | 工作中 | Agent 正在执行任务 |
| 💤 | 空闲 | Agent 可以接收新任务 |
| 💀 | 离线 | Agent 不可用 |

## 🧪 测试日志输出

### 方法 1：使用演示脚本

```bash
./scripts/test-log-demo.sh
```

### 方法 2：手动创建并调度任务

```bash
# 1. 创建任务
TASK=$(curl -s -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试日志输出",
    "assignedTo": "niuma-searcher"
  }' | jq -r '.task.id')

# 2. 调度任务
curl -X POST http://localhost:3000/api/tasks/dispatch \
  -H "Content-Type: application/json" \
  -d "{
    \"taskId\": \"$TASK\",
    \"waitForIdle\": true,
    \"maxWait\": 300000,
    \"checkInterval\": 10000
  }"
```

## 📈 日志分析

### 监控等待时间

日志中显示每次检查的时间：

```
🔍 [检查 #1] 14:30:25 - 已等待 0.0 分钟 (剩余 30.0 分钟)
🔍 [检查 #2] 14:30:55 - 已等待 0.5 分钟 (剩余 29.5 分钟)
🔍 [检查 #3] 14:31:25 - 已等待 1.0 分钟 (剩余 29.0 分钟)
```

可以根据这些信息：
- 计算平均等待时间
- 识别高峰时段
- 优化检查间隔

### 识别问题模式

**Agent 长时间忙碌**：
```
🔍 [检查 #1-N] 状态: 🔨 working (持续很长时间)
```
→ 可能需要：增加 Agent 数量或优化任务分配

**频繁离线**：
```
🔍 [检查 #N] 状态: 💀 offline
```
→ 可能需要：检查 Agent 稳定性或网络连接

**子任务堆积**：
```
当前任务: 5+ 个子任务正在执行
```
→ 可能需要：限制并发数或优化任务拆分
